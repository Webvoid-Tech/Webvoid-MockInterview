# 🎓 AI Mock Interview Agent

An intelligent AI agent built with **Google ADK (Agent Development Kit)** and **Groq (via LiteLLM)** that conducts technical mock interviews, evaluates answers, and provides constructive feedback to help you prepare for AI/ML interviews.

## 🌟 Features

- **Built with Google ADK**: Leverages Google's official Agent Development Kit framework
- **Hybrid Architecture**: Gemini 2.0 Flash for agent (ADK compatible) + Groq Llama3 for tools (ultra-fast)
- **Best of Both Worlds**: Reliable orchestration + blazing fast tool execution
- **Intelligent Question Generation**: Generates real interview questions using Groq LLM
- **Answer Evaluation**: Analyzes your responses using Groq LLM with detailed scoring
- **Constructive Feedback**: Offers actionable tips generated by Groq LLM
- **Adaptive Difficulty**: Adjusts question difficulty based on your performance
- **Conversational Interface**: Natural conversation flow with session management
- **Multiple Interfaces**: Choose between CLI or ADK's built-in web interface

## 🏗️ Architecture

This application demonstrates **Google ADK's LlmAgent** architecture with:

### Core Components

1. **LlmAgent**: The main agent orchestrator with instructions and tools
2. **Runner**: Manages agent execution and session state
3. **Tools**: Three specialized function tools that the agent can call

### Agent Tools

The agent is equipped with three function tools:

1. **`generate_interview_question`**: Uses Groq's Llama 3.3 70B to create realistic interview questions
2. **`evaluate_student_answer`**: Uses Groq's Llama 3.3 70B to score answers and identify strengths/weaknesses
3. **`generate_feedback`**: Uses Groq's Llama 3.3 70B to provide personalized tips and recommendations

**All tools use LiteLLM** to call Groq's API, ensuring fast and high-quality responses.

### Agent Workflow

```
Start Interview
      ↓
Agent uses generate_interview_question tool
      ↓
Student Answer
      ↓
Agent uses evaluate_student_answer tool
      ↓
Agent uses generate_feedback tool
      ↓
Provide Feedback
      ↓
Decide Next Question (adaptive difficulty)
      ↓
Repeat
```

### ADK Architecture

```
Student Input
      ↓
Runner (Session Management)
      ↓
LlmAgent (Groq Llama 3.3 70B via LiteLLM)
  - Instructions (Agent behavior)
  - Tools (Function calling)
      ↓
Tool Execution (Groq Llama 3.3 70B via LiteLLM)
  - generate_interview_question
  - evaluate_student_answer
  - generate_feedback
      ↓
Agent Response
```

## 🚀 Installation

1. **Clone or navigate to the project directory**

```bash
cd MockInterviewAgent
```

2. **Install Google ADK and dependencies**

```bash
pip install -r requirements.txt
```

**Note**: This project uses:
- **Google ADK** (`google-adk`) - Official Agent Development Kit for agent orchestration
- **LiteLLM** - Unified interface for calling Groq's API
- **Groq** - Ultra-fast inference for Llama 3.3 70B (both agent and tools)

3. **Set up your API Keys**

Create a `.env` file in the project root:

```bash
cp .env.example .env
```

Edit `.env` and add **both** API keys:

```
GOOGLE_API_KEY=your_google_api_key_here
GROQ_API_KEY=your_groq_api_key_here
```

**To get a Google API key:**
- Visit [Google AI Studio](https://aistudio.google.com/apikey)
- Create a new API key for Gemini
- **Required** for the main agent (ADK compatibility)

**To get a Groq API key:**
- Visit [Groq Console](https://console.groq.com/keys)
- Sign up for a free account
- Create an API key
- Used for ultra-fast tool execution (800+ tokens/sec!)

## 💻 Usage

### Option 1: ADK Web Interface (Recommended)

Run the ADK built-in web interface:

```bash
adk web
```

This will:
1. Discover the `mock_interview_agent` folder
2. Load `agent.py` which defines `root_agent`
3. Start a web server at `http://localhost:8000`

The web interface provides:
- Clean chat interface
- Real-time agent responses
- Automatic tool calling
- Session management
- Message history
- Event inspection

**How it works:**
- ADK discovers the `mock_interview_agent` module
- Loads the `root_agent` from `agent.py`
- Tools are defined as Python functions in the same file
- The web UI provides a full-featured chat interface
- All state is managed by ADK

**Custom port:**
```bash
adk web --port 8080
```

### Option 2: Command Line Interface

Run the CLI version:

```bash
python cli_app.py
```

Follow the prompts to:
1. Select your interview topic
2. Choose difficulty level
3. Have a natural conversation with the AI interviewer
4. Receive real-time feedback
5. Continue until you end the interview

**How it works:**
- Uses async/await for agent communication
- Streams responses from the agent
- Maintains conversation context automatically

## 🔧 Key ADK Concepts Used

### 1. LlmAgent with Instructions

The agent is configured with detailed instructions that guide its behavior:

```python
INTERVIEW_AGENT_INSTRUCTION = """You are an expert AI Mock Interview Agent...
- Use the `generate_interview_question` tool to create questions
- Use the `evaluate_student_answer` tool to analyze responses
- Use the `generate_feedback` tool to provide constructive feedback
...
"""
```

### 2. Function Tools

Tools are defined as simple Python functions with docstrings:

```python
def generate_interview_question(topic: str, difficulty: str, previous_questions: str = "") -> str:
    """Generates a technical interview question based on topic and difficulty.
    
    Args:
        topic: The interview topic (e.g., 'AI/ML', 'Machine Learning')
        difficulty: The difficulty level ('easy', 'medium', or 'hard')
        previous_questions: Comma-separated list of previously asked questions
    
    Returns:
        JSON string with question, difficulty, and topic fields
    """
    # Tool implementation
```

The ADK automatically:
- Parses the function signature
- Uses the docstring for tool description
- Handles JSON serialization
- Manages tool calling

### 3. Runner and Session Management

```python
from google.adk.runners import Runner
from google.adk.sessions import InMemorySessionService

session_service = InMemorySessionService()
runner = Runner(agent=agent, app_name="mock_interview_app", session_service=session_service)
```

The Runner:
- Manages conversation state
- Handles tool execution
- Maintains session history
- Provides async streaming

## 📝 API Usage Example

```python
import asyncio
from agent import create_interview_agent, create_runner
from google.genai import types

async def run_interview():
    # Create agent and runner
    agent = create_interview_agent()
    runner = create_runner(agent)
    
    # Start interview
    message = types.Content(
        role='user',
        parts=[types.Part(text="Start a mock interview on AI/ML at medium difficulty")]
    )
    
    # Get response
    async for event in runner.run_async(
        user_id="student_001",
        session_id="session_001",
        new_message=message
    ):
        if event.is_final_response() and event.content:
            print(event.content.parts[0].text)

asyncio.run(run_interview())
```
